Azure Policy —
håndhev regler automatisk

Satt opp tre Azure Policies som håndhever tagging, begrenser ressurser til Norway East og kontrollerer tillatte VM-størrelser. Det som skiller et tilfeldig lab-miljø fra et gjennomtenkt ett.

Fullført Azure Policy Governance AZ-104 pensum Offentlig sektor

Hva er Azure Policy og hvorfor bruker vi det?

Azure Policy er et regelsett som automatisk håndhever at ressurser i Azure følger organisasjonens krav. I stedet for å stole på at folk husker å gjøre ting riktig, setter du regler som evalueres kontinuerlig mot alle ressurser.

Deny
Blokkerer opprettelse av ressurser som bryter regelen. Ingen vei rundt — ressursen kan ikke opprettes.
Audit
Ressursen opprettes, men merkes som non-compliant i Defender for Cloud. Brukes for å kartlegge uten å blokkere.
Modify / Append
Legger automatisk til tags eller innstillinger når en ressurs opprettes — uten å avvise den.
DeployIfNotExists
Deployer automatisk en ressurs hvis den mangler — f.eks. aktivere Defender for Cloud på nye subskripsjoner.
Policy 1 — Krev tags på ressurser Fullført
1
Tildel built-in policy: Require a tag on resources
Azure har hundrevis av innebygde policies. For tagging bruker vi den innebygde policyen "Require a tag on resources" — ingen grunn til å skrive egne regler når Microsoft allerede har laget dem.
💡
Scope: Vi setter policyen på ressursgruppe-nivå (rg-lab-01), ikke på subskripsjons-nivå. I produksjon ville du satt det høyere opp, men i lab holder ressursgruppen.

Scoperg-lab-01Gjelder kun ressurser i lab-ressursgruppen

⚠️
Deny-effekt betyr at eksisterende ressurser uten taggen ikke slettes — de merkes bare som non-compliant. Deny gjelder kun nye ressurser som opprettes etter policyen er aktiv.
Policy 2 — Kun Norway East Fullført
1
Tildel built-in policy: Allowed locations
I offentlig sektor er dataplassering kritisk — persondata og sensitive opplysninger skal ikke havne utenfor EØS uten særskilt vurdering. Denne policyen blokkerer opprettelse av ressurser utenfor Norway East.

Retting — Tag eksisterende ressurser med PowerShell Fullført
1
Installer Az-modulen og sett Execution Policy
For å tagge alle ressurser på én gang ble PowerShell brukt. Az-modulen var ikke installert fra før, og Windows blokkerte kjøring av scripts som standard.
⚠️
Execution Policy er en Windows-sikkerhetsinnstilling som blokkerer kjøring av PowerShell-scripts. Dette er en vanlig feil første gang man bruker PowerShell på en ny maskin.
# Installer Az-modulen (én gang, tar 5-10 minutter) Install-Module -Name Az -Repository PSGallery -Force -AllowClobber -Scope CurrentUser # Fiks Execution Policy — blokkerer scripts som standard i Windows Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser # Last inn modulen og logg inn Import-Module Az.Accounts Connect-AzAccount
2
Tagg alle ressurser i rg-lab-01 med PowerShell
Scriptet henter alle ressurser i ressursgruppen, beholder eksisterende tags og legger til de nye. dcr-lab-01 (Data Collection Rule) støtter ikke tagging via Set-AzResource og ble tagget manuelt i portalen etterpå.
# Definer ressursgruppe og tags $resourceGroupName = "rg-lab-01" $tags = @{ "environment" = "lab" "owner" = "Karsten" "purpose" = "training" } # Hent alle ressurser i ressursgruppen $resources = Get-AzResource -ResourceGroupName $resourceGroupName # Loop gjennom og tagg hver ressurs foreach ($resource in $resources) { $existingTags = $resource.Tags if ($existingTags -eq $null) { $existingTags = @{} } $mergedTags = $existingTags + $tags Set-AzResource -ResourceId $resource.ResourceId -Tag $mergedTags -Force Write-Host "Tagget: $($resource.Name)" } Write-Host "Ferdig — alle ressurser er tagget"
💡
Vi bruker $mergedTags = $existingTags + $tags i stedet for å overskrive direkte. Dette sikrer at eksisterende tags ikke slettes — viktig i produksjonsmiljøer der ressurser kan ha tags fra andre systemer.
3
Tving compliance-evaluering og verifiser
Azure Policy evaluerer ikke ressurser umiddelbart. Etter tagging ble en manuell compliance-scan trigget via PowerShell, og alle 9 ressurser ble verifisert som compliant.
# Tving en ny compliance-evaluering (tar 5-15 minutter) Start-AzPolicyComplianceScan -ResourceGroupName "rg-lab-01"
💡
Remediation task er grået ut på Deny-policies — remediation støttes kun for Modify og DeployIfNotExists. For Deny-policies brukes Start-AzPolicyComplianceScan for å tvinge re-evaluering etter at ressurser er rettet opp manuelt.
