Azure Monitor
Alert Rules

Satt opp en metric alert rule på CPU-bruk for vm-lab-01, med e-postvarsling via Action Group. Inkluderte bruk av Policy Exemption for å omgå Norway East-policyen for globale ressurser.

Fullført Azure Monitor Log Analytics rg-lab-01 AZ-104 pensum

// Hva er Azure Monitor?

Azure Monitor samler inn metrikker og logger fra Azure-ressurser og lar deg sette opp varsler når noe er unormalt. Det er sentralt i AZ-104 og er grunnlaget for god driftsovervåking i produksjon.

Alert Rule
Definerer hva som skal overvåkes, hvilken terskel som trigger varselet, og med hvilken alvorlighetsgrad.
Action Group
Hvem som varsles og hvordan — e-post, SMS, webhook, Azure Function, Logic App og mer.
Metric vs Log Alert
Metric alerts er rask-respons på tall (CPU, minne). Log alerts kjører KQL-spørringer mot Log Analytics og er mer fleksible.
Severity
Sev 0 (Critical) → Sev 4 (Verbose). Sev 2 Warning passer for ressursovervåking i lab og ikke-kritisk produksjon.
Steg 1 — Opprett Alert Rule Fullført
1
Konfigurer alert rule på vm-lab-01
Navigerte til vm-lab-01 → Monitoring → Alerts → + Create → Alert rule. Valgte CPU som metrikk og satte terskelen til 80%.
# Alert rule — innstillinger Navn: alert-cpu-vm-lab-01 Scope: vm-lab-01 Condition: Percentage CPU > 80 Severity: 2 — Warning Region: Norway East
💡
Alert rules for metrikker lagres i samme region som ressursen de overvåker — i dette tilfellet Norway East. Action Groups er derimot globale ressurser og følger ikke lokasjons-policyen din.
Steg 2 — Policy Exemption for Action Group Fullført
1
Håndtere Norway East-policyen
Action Groups er en global ressurs i Azure Monitor og kan ikke opprettes i en spesifikk region. "Allow Norway East only"-policyen blokkerte opprettelsen. I stedet for å deaktivere policyen midlertidig ble det opprettet en Policy Exemption — en mer presis og sporbar løsning.
💡
Exemption vs Disabled: En exemption unntar én spesifikk ressurs eller scope fra policyen uten å påvirke resten. Deaktivering av policyen fjerner beskyttelsen for hele subscription-scopet midlertidig. Exemption er alltid å foretrekke i produksjon.
InnstillingVerdiForklaring
Exemption scopealert-cpu-vm-lab-01Unntar kun denne ressursen
PolicyAllow Norway East onlyPolicyen som blokkerte
KategoriWaiverMidlertidig unntak — ikke en permanent feil i policyen
⚠️
I produksjon bør exemptions dokumenteres med begrunnelse og eventuell utløpsdato. Azure Policy støtter expiry-dato på exemptions — bruk det i offentlig sektor der sporbarhet er viktig.
Steg 3 — Opprett Action Group Fullført
1
Konfigurer e-postvarsling
Opprettet Action Group under opprettelsen av alert rule. Valgte Global som region — dette er korrekt og forventet for Action Groups uavhengig av lokasjons-policy.
# Action Group — innstillinger Navn: ag-lab-notifications Display name: lab-notify Region: Global Notification: Email/SMS/Push/Voice Navn: epost-varsel
Steg 4 — Koble Action Group til Alert Rule Fullført
1
Knytte varsling til regelen
Action Group ble koblet til alert rule via Edit → Actions → Select action groups. Alert rule var opprettet uten action group i første omgang, og ble oppdatert i etterkant.
💡
En Action Group kan gjenbrukes på tvers av mange alert rules. Neste gang du oppretter en alert rule kan du velge ag-lab-notifications direkte — du trenger ikke opprette ny Action Group.
